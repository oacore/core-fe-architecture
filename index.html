Key Architectural Decisions

This structure is built on core principles that enhance scalability and maintainability for client-side applications. These decisions ensure that the app can grow in complexity without sacrificing organization or developer velocity.

`src/` Directory

Organizing application code within a `src` directory is a standard practice that keeps the project root clean and separates source code from configuration files.

Feature-Based Organization

This structure uses a `features/` directory to contain all the code related to a specific feature (components, hooks, state), improving maintainability and scalability.

Client-First Logic

All logic is handled on the client-side, with dedicated `services/` for REST API interaction. This is ideal for applications that rely on external data sources and do not require server-side rendering for every page.
